- В Дзене можно вручную удалить тестовые черновики с заголовками `Тест картинки Дзен — проверка загрузки`.
- В Дзене можно проверить опубликованную статью `Как устроен вирус`: она была создана автоматическим запуском после последних исправлений.
- Не публиковать и не пересылать значения токенов из `.env` и Railway-логов.

## Текущие риски

- Интерфейс Дзена может измениться, поэтому Playwright-селекторы нужно проверять при новых сбоях.
- Если сессия Дзена устареет, нужно обновить `DZEN_STORAGE_STATE_JSON` в Railway.
- Railway-логи могут показывать URL Telegram Bot API с токеном; при подозрении на компрометацию токен лучше перевыпустить.
- При локальном ручном запуске без предварительного `init_db()` публикация успела пройти, но запись статуса в локальную SQLite завершилась ошибкой `no such table: dzen_publications`; production FastAPI вызывает `init_db()` при старте.
