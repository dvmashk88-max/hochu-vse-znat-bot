## Что еще нужно сделать

### 1. Добавить тематическую картинку в Дзен

Сейчас картинка отправляется только в Telegram. В Дзен передается заголовок и текст, но не изображение.

Нужная логика:

- В `publisher.py` уже получается `image_bytes` для Telegram.
- Эти же `image_bytes` нужно передать в `publish_draft`.
- `publish_draft` нужно расширить параметром `image_bytes: bytes | None`.
- В `dzen_publisher.py` нужно добавить вставку изображения в редактор Дзена.

Возможные способы вставки:

1. Сохранить `image_bytes` во временный файл.
2. В редакторе Дзена найти кнопку добавления изображения.
3. Через Playwright использовать `set_input_files()` для скрытого `input[type=file]`.
4. Дождаться, что изображение загрузилось.
5. Только после этого нажимать `Далее` и `Опубликовать`.

Важно:

- Временный файл должен удаляться после публикации.
- Если картинка не загрузилась, статья не должна падать полностью: лучше опубликовать текст и записать warning в лог.
- Перед автопубликацией нужно проверять, что в редакторе есть текст и, если возможно, что картинка загружена.

### 2. Сделать картинки более цепляющими

Тематическая картинка должна не просто совпадать с темой, а цеплять внимание.

Нужное направление:

- Добавить отдельную функцию генерации поискового запроса для визуала:
  - не только `GPS satellite navigation technology map`;
  - а более выразительно: `dramatic GPS satellites orbiting earth navigation technology`.
- Для науки и космоса использовать запросы с визуальным эффектом:
  - `dramatic`;
  - `macro`;
  - `cinematic`;
  - `high contrast`;
  - `space`;
  - `laboratory`;
  - `close up`.
- Избегать скучных запросов вроде `education learning knowledge`.
- Для тем, где фотостоки дают слабые картинки, рассмотреть генерацию изображения через AI и загрузку его в Telegram/Дзен.

### 3. Улучшить качество выбора изображения

Сейчас выбирается случайный вариант из первых 10.

Лучше сделать:

- брать 10-20 результатов;
- отсеивать неподходящие по описанию/alt-тексту;
- не использовать маленькие или вертикальные изображения;
- хранить историю URL картинок, чтобы не повторять одно и то же;
- логировать тему, поисковый запрос и источник картинки.

### 4. Добавить обложку/первую картинку в Дзен как обязательный шаг

Для Дзена картинка должна быть не просто внутри статьи, а в начале материала или как обложка, если редактор это позволяет.

Проверить в интерфейсе Дзена:

- есть ли отдельная кнопка обложки;
- можно ли вставить изображение первым блоком статьи;
- как Дзен выбирает превью публикации;
- нужно ли после загрузки картинки ждать crop/preview editor.

### 5. Добавить тестовый режим без публикации

Чтобы спокойно проверять Дзен, нужен режим:

- создать черновик;
- вставить заголовок;
- вставить текст;
- вставить картинку;
- не нажимать `Опубликовать`.

Это позволит тестировать загрузку картинки без риска выпустить тестовую статью.

## Риски

- Интерфейс Дзена может меняться, поэтому Playwright-селекторы нужно периодически проверять.
- Всплывающие окна Дзена могут появляться снова после обновлений интерфейса.
- Если сессия Дзена устареет, `DZEN_STORAGE_STATE_JSON` нужно будет обновить.
- Railway-логи и CLI могут показывать секреты. Токены нельзя пересылать в открытом виде после настройки.
- Telegram-токен уже был показан в переписке и логах, поэтому при подозрении на компрометацию его лучше снова перевыпустить.

## Следующий практический шаг

После проверки ближайшей автоматической публикации:

1. Если Telegram и Дзен публикуются корректно, добавить передачу картинки в `publish_draft`.
2. Реализовать загрузку изображения в редактор Дзена через Playwright.
3. Протестировать в режиме черновика.
4. После успешного черновика включить проверку боевой автопубликации по таймеру.
