## Telegram

- Telegram-публикация работает через `python-telegram-bot`.
- Канал берется из `TELEGRAM_CHANNEL_ID`.
- Если `image_bytes` есть, бот отправляет фото с подписью.
- Если картинки нет, бот отправляет текстовую публикацию.
- Публикация запускается общим циклом `app/publisher.py` по расписанию APScheduler.
- Известный риск: HTTP-логи библиотек могут показывать URL Telegram Bot API с токеном, если логирование `httpx` остается на INFO.

## Dzen

- Дзен работает через Playwright в `app/dzen_publisher.py`.
- Авторизация в production берется из Railway-переменной `DZEN_STORAGE_STATE_JSON`.
- Локальный fallback для сессии: `storage/dzen_cookies.json` (файл в `.gitignore`).
- `DZEN_AUTO_PUBLISH=true` в Railway production, поэтому Дзен публикует статьи автоматически.
- Картинка загружается в редактор Дзена через кнопку загрузки файла и вставляется перед текстом.
- После вставки картинки бот ждет подтверждение загрузки и автосохранение.
- Если картинка не передана или загрузка не подтвердилась, бот пишет warning и может продолжить публикацию текста.
- Известные проблемы:
  - интерфейс Дзена может меняться, Playwright-селекторы нужно перепроверять при сбоях;
  - если Dzen-сессия устареет, нужно обновить `DZEN_STORAGE_STATE_JSON`;
  - настройка обсуждений/комментариев пока не автоматизирована.

## MAX

- MAX-публикация подключена через `app/max_publisher.py`.
- Railway variables: `MAX_BOT_TOKEN`, `MAX_CHANNEL_ID`.
- Бот отправляет текст статьи в канал MAX.
- Если есть `image_bytes`, бот загружает изображение через MAX upload API и отправляет attachment.
- Если MAX отвечает, что attachment еще не готов, выполняется повторная отправка с паузой.
- Ошибка MAX записывается отдельно и не должна останавливать Дзен.

## VK

- VK-публикация подключена через `app/vk_publisher.py`.
- Railway variables: `VK_ACCESS_TOKEN`, `VK_GROUP_ID`.
- `VK_GROUP_ID=239439853`, публикация идет в сообщество `@factdaily`.
- Текстовые посты в VK работают через `wall.post`.
- Текущий `VK_ACCESS_TOKEN` является токеном сообщества.
- Код пытается загрузить картинку в VK, если `image_bytes` есть.
- Известное ограничение: `photos.getWallUploadServer` не работает с group token и отвечает ошибкой group auth.
- В коде есть fallback: если загрузка картинки в VK падает, пост публикуется текстом.
- В Railway также есть `VK_APP_ID`, `VK_CLIENT_SECRET`, `VK_REDIRECT_URI`; текущий production-код их не использует.
- Что еще не реализовано:
  - отдельный user access token администратора для загрузки картинок в VK;
  - использование user token для `photos.getWallUploadServer` и `photos.saveWallPhoto`;
  - публикация VK attachments вида `photo<owner_id>_<photo_id>`.

## Генерация статей

- Генерация текста находится в `app/generator.py`.
- API: OpenRouter Chat Completions.
- Основная модель: `openai/gpt-4o-mini`.
- После основной модели подключен fallback-каскад OpenRouter-моделей.
- Fallback-хвост состоит только из бесплатных моделей `:free` или `openrouter/free`.
- В коде есть проверка, что `openai/gpt-4o-mini` стоит первой, а платные модели не попадают в fallback-хвост.
- Prompt-файл: `prompts/post_prompt.txt`.
- Основной режим генерации текста использует `max_tokens=1200`, `temperature=0.8`.
- Для генерации темы используется отдельный prompt в `app/generator.py`, `max_tokens=350`, `temperature=1.0`.
- Перед статьей бот сначала генерирует тему-кандидат в JSON: `title`, `category`, `angle`, `keywords`.
- Тема проверяется на похожесть с последними публикациями и на слишком популярные/избитые научпоп-фрагменты.
- Если AI-тема не проходит проверку или OpenRouter недоступен, используется fallback-пул тем в `app/topic_selector.py`.
- Fallback-пул тем расширен со 101 до 151 темы.
- Для защиты от повторов используется история последних 50 публикаций.
- В `published_topics` добавлены локальные поля метаданных темы: `category`, `angle`, `keywords`.
- После успешной публикации `app/publisher.py` сохраняет тему и метаданные темы в базу.
- Лимит длины статьи усилен: целевой диапазон 800-1200 символов.
- Если модель выходит за диапазон, код просит модель сократить/дополнить текст.
- Если после повторных попыток текст все еще длиннее 1200 символов, включается локальный hard trim по предложению с сохранением хэштегов.
- В `prompts/post_prompt.txt` добавлены ограничения против выдуманных имен, дат, экспериментов, источников и точных чисел.

## Картинки

- Текущая рабочая схема картинок: `Pexels -> Pixabay`.
- Логика находится в `app/images.py`.
- `app/publisher.py` получает `image_bytes` через `fetch_image(generate_image_query(topic))`.
- Те же `image_bytes` передаются в Telegram, MAX, VK и Дзен.
- Если Pexels ничего не вернул, код пробует Pixabay.
- Если оба источника не дали картинку или загрузка упала, публикация может продолжиться без изображения.
- Эксперимент Hugging Face / FLUX / AI-cover / cover_composer закрыт и полностью удален из проекта.
- В коде, `.env`, `.env.example` и Railway больше нет `HF_TOKEN`, `AI_COVER_ENABLED`, `AI_COVER_HF_PROVIDER`, `AI_COVER_HF_MODEL`.
- Проект сейчас работает на старом механизме Pexels/Pixabay.
